# 工程表への休日設定機能の導入計画

工程表（メイン画面）において、特定の日付を休日として設定し、全ての担当者の行を跨ぐ垂直なバー（縦線）を表示する機能を実装します。

## ユーザーレビューが必要な点
- **操作方法**: 日付ヘッダーをクリックすることで、その「曜日」自体の休日設定をオン/オフできるようにします。
- **永続性**: ある週で月曜日を休日にすると、翌週以降の全ての月曜日も休日として扱われます。
- **表示**: これまで通り、休日設定された曜日の列が垂直に強調表示されます。

## 変更計画

### 1. 状態管理の変更 (app.js)
- `this.state.holidays` を「日付（YYYY-MM-DD）」の配列から「曜日インデックス（0:月〜6:日）」の配列へ変更します。
- `validateIntegrity` で古い形式（文字列）が含まれている場合にリセットする処理を追加します。

### 2. トグルロジックの更新 (app.js)
- `toggleHoliday` がインデックスを受け取るようにし、0〜6の数値を管理するようにします。

### 3. 描画ロジックの更新 (app.js)
- `renderGrid` 内で、日付の文字列比較ではなく、ループのインデックス `i` を直接 `state.holidays` と照合するように簡略化します。

### 4. スタイルの定義 (style.css)
- `.holiday-bar` のスタイル（半透明の赤背景、点線、z-index調整など）を追加します。
- ヘッダー部分のホバー効果（クリック可能であることを示す）を追加します。

## 検証プラン

### 自動テスト (ブラウザ)
- [ ] 日付ヘッダーをクリックして、赤い縦バーが表示されることを確認。
- [ ] 再度クリックして、バーが消えることを確認。
- [ ] 週を切り替えても、設定した日付のバーが正しく維持・表示されることを確認。
- [ ] データをJSON出力し、再読み込みした際に休日設定が復元されることを確認。

### 手動確認
- バーが工程表のタスク（バー）の邪魔にならず、かつ視認性が高いことを確認。
