# ウォークスルー：週別データ保存機能の仕様変更

ユーザー様からの新しいご要望に基づき、以下の仕様変更を行いました。

## 1. 変更内容一覧

| 項目 | 以前の挙動 | 変更後の挙動 |
|---|---|---|
| **全データの週固有化** | `teams`, `staff`, `projects` などを共有データとし、`schedule`, `holidays` のみ週固有としていた | **すべてのデータ**（チーム、スタッフ、プロジェクト、タグ、スケジュール、休日設定等すべて）を週固有データとし、週ファイルを読み込んだ時に `app.state` を完全に上書きする |
| **引き継ぎダイアログキャンセル時** | 週固有データをクリアして新規の空週データファイルを作成し、週を変更していた | 新規ファイルは作成せず、**週の切り替え処理自体をキャンセルして元の週にとどまる** |
| **データ全削除機能** | 画面上に「データ全削除」ボタンがあり、`app.clearAllData()` により初期化可能だった | **ボタン自体をUIから削除**し、`app.js` からも関連ロジックを完全に削除 |

---

## 2. 具体的な変更箇所

### A. 全てのデータを週データ固有化
`js/sync-manager.js` の `_applyWeekData(mondayISO)` において、マージを行っていた処理を以下のように単純な上書き処理へ変更しました：
- ファイルが存在する場合は、`Object.assign(app.state, data)` でロードした全てのデータ（チーム、スタッフ、プロジェクト等すべて）を state に上書き適用します。
- 切り替え先でデータがまだ存在しない（かつ引き継がないまま強制的に初期表示される）ような例外的なケースでは、全てのデータを空（`[]`）にして初期化します。

### B. 引き継ぎダイアログキャンセル時の週変更中止
`js/sync-manager.js` の `saveCurrentWeekAndLoadNew` は成否を `boolean` で返すように変更しました。
- 切り替え先ファイルが存在しない際、確認ダイアログ（`confirm`）でキャンセルされた場合は `false` を返して終了します。
- `app.js` の `moveWeek(n)` では、戻り値が `false`（キャンセル）だった場合、`app.state.monday` は変更されず、そのまま現在の週で再描画（`this.render()`）されます。

### C. データ全削除機能の削除
- `index.html` のヘッダー右側にあった `<button class="btn btn-danger" onclick="app.clearAllData()">データ全削除</button>` を削除しました。
- `app.js` の `clearAllData()` 関数を削除しました。

### D. 操作ボタンの左側空白解消と色分け
- **アイコンの不要化**:
  - `更新` ボタンおよび `クイック同期` ボタンから `<i>` タグを削除し、ロード失敗などによるテキスト左側の不要な空白（隙間）が発生しないように修正しました。
- **わかりやすい色分けの適用**:
  - `更新` ボタン: 緑 (`btn-success`)
  - `クイック同期` ボタン: 黄/オレンジ (`btn-warning` クラスを追加定義)
  - `同期設定` ボタン: 落ち着いたグレー (`btn-secondary` クラスを追加定義)
  - `印刷` ボタン: 水色 (`btn-info` クラスを追加定義)
  - `管理画面` ボタン: 紫/インディゴ (`btn-indigo` クラスを追加定義)
  - `製番リスト (別窓)` ボタン: 青 (`btn-primary`)
  - それぞれ異なるカラー of クラスを [style.css](file:///c:/Users/kohei/.gemini/antigravity-ide/scratch/web_project_manager/style.css) に定義し、ヘッダーに配置されているボタンを直感的に区別しやすいように色分けしました。

### E. FHD向け全体UI・文字サイズの最適化（縮小）
- **ベースフォントサイズの縮小**:
  - `html` 要素に対して `font-size: 14px` (従来のブラウザ標準16pxから縮小) を設定し、`rem` 単位で指定されたUI全体の文字サイズや余白を一括でコンパクト化しました。
- **グリッド・レイアウト寸法の縮小**:
  - グリッドの行の高さ (`--row-height`) を `120px` から `95px` に縮小。
  - 左側サイドバー（担当者名欄）の横幅 (`--sidebar-width`) を `240px` から `200px` に縮小。
  - `grid-header`（曜日・日付ヘッダー）の縦幅を `64px` から `48px` に縮小。
- **余白 (Padding/Gap) の調整**:
  - `header` のパディングを `0.75rem 2rem` から `0.5rem 1.5rem` に変更。
  - `week-nav` (週のナビゲーション) のパディングを `1.5rem 2rem` から `0.75rem 1.5rem` に、ギャップを `2rem` から `1.5rem` にそれぞれ縮小。
  - `week-text` の最小幅を `320px` から `280px` に、フォントサイズを `1.4rem` から `1.2rem` に縮小。
  - 左側サイドバー `grid-sidebar` のパディングを `1rem` から `0.5rem 0.75rem` に、スタッフ名フォントを `1.05rem` から `0.95rem` へ縮小。
- **目的**:
  - FHD (1920x1080) 解像度においてスクロールすることなく、より多くの行（スタッフ人数）やデータを一度に俯瞰できるようにしました。

### F. メンバーの「工数/日」パラメータ追加
- **データ構造の拡張 (`workHoursPerDay`)**:
  - メンバー（スタッフ）オブジェクトに `workHoursPerDay` プロパティを追加しました。
  - 既存のデータ整合性チェック (`validateIntegrity`) において、未定義のメンバーには自動でデフォルト値である `8.00` がセットされます。
- **管理画面 (モーダル) のUI拡張**:
  - メンバー追加フォームに「工数/日」の数値入力欄を追加しました（追加時の初期値は `8.00`）。
  - メンバー一覧テーブルに「工数/日」の列を追加し、各メンバーの工数をその場で編集できるようにしました（小数第2位までの入力に対応）。
  - 値が更新された際には、浮動小数点数に整形して `workHoursPerDay` に保存する変更関数 `upStaffH()` を追加しました。
  - **0.25単位での変更制限**:
    - 入力フォームの `step` 属性を `0.25` に設定し、ブラウザの矢印キーやスピンボタンによる値の増減が `0.25` 刻みで行えるようにしました。
    - キーボードによる任意の手入力に対しても、内部ロジック（`Math.round(val * 4) / 4`）により自動的に `0.25` 単位に四捨五入して丸め処理が行われるように実装しました。
- **メイン画面への表示追加**:
  - 工程表の左側サイドバー (`grid-sidebar`) にあるチーム名表示の右隣に、各メンバーの設定された工数を `(8.00h/日)` のように小さく併記して表示するようにしました。

### G. 担当者行の動的サイズ（高さ）変更
- **基本の高さの最適化 (2行分化)**:
  - 新たな案件をセルクリックで割り振る際のマウス操作性を確保するため、デフォルト（案件未登録または1段表示のみ）の行の高さ (`--row-height`) を **`83px`**（2行分に相当）としました。
- **重なり検知時の動的拡張（1行分の空きを常時確保）**:
  - `app.js` の `drawBars` 描画処理において、複数の案件が重複して割り振られた場合（`maxLv >= 1`）、表示されるバーの段数に加えて常に「+1行分（35px）」の空きスペースを確保するよう拡張ロジックを変更しました。
  - 計算式を `(maxLv + 2) * 35 + 13` px にしたことで、2段重複時は `118px`（3行分）、3段重複時は `153px`（4行分）に自動拡張されます。
  - これにより、何段重ねになっても最下部に必ず1行分のクリック可能な空き領域が維持されるため、新規案件のドラッグ＆ドロップや追加クリック時の操作性が常に良好に保たれます。
