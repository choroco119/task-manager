# 印刷専用レイアウト (Print View) の実装計画

工程表を紙やPDFとして綺麗に出力するための、印刷専用レイアウト機能を実装します。

## ユーザー確認事項
> [!IMPORTANT]
> 印刷時に、画面下部の「統計エリア（各チームの合計工数など）」も印刷に含めますか？
> 現時点では以下の要素を **非表示** にする予定です：
> - ヘッダー（データ出力、管理画面ボタンなど）
> - 週移動ボタン（前週・次週）
> - 詳細情報パネル（右側のサイドバー）
> - 割り当てのヒント（📍...）
> - ゴミ箱や各種検索フィルター

## 仕様案

### 1. 印刷開始ボタンの追加
ヘッダーの「データ出力」の隣に「印刷用画面」ボタンを追加します。クリックすると、ブラウザの印刷ダイアログを起動します。

### 2. 印刷用スタイル (@media print) の定義
専用のCSSを `style.css` に追加します。
- **背景と配色**: 背景を白（透明）、文字を黒に強制し、視認性を高めます。
- **レイアウトの最適化**: 
  - A4横（Landscape）での出力を想定し、グリッドの幅を100%に広げます。
  - スティッキーヘッダー（固定表示）などの印刷時に邪魔になる制御を解除します。
- **要素の制御**: 印刷に不要なボタン類、検索バー、影（box-shadow）などをすべて無効化します。

### 3. 特殊要素の追加
印刷時のみ表示されるタイトル（例：「工程管理スケジュール (2026/04/06 - 2026/04/12)」）を最上部に追加します。

## 実施手順 (app.js / style.css)
1. `index.html` に印刷ボタンを追加し、`app.print()` メソッドを呼び出すように設定。
2. `app.js` に `print()` メソッドを実装（単純な `window.print()` の呼び出しが基本）。
3. `style.css` に大量の `@media print` 定義を追加し、印刷時の「白背景・黒文字・不要UI非表示」を実現。
