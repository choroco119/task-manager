# 削除案件の復元機能（ゴミ箱機能）の導入計画

一度削除した製番データを「ゴミ箱」に一時保存し、必要に応じて元のリストへ戻せるようにします。

## ユーザーレビューが必要な点
- **復活時の番号**: 案1「リストの最後尾に新しい連番で追加」を採用します。
- **データ保持**: 番号（No.）以外のすべての情報（指令書、客先、担当者、タグ、備考など）は削除時の状態を完全に保持して復元します。
- **削除日時の記録**: 削除実行時の日時を自動的に記録し、ゴミ箱内で確認できるようにします。
- **保持件数制限**: 快適な動作を維持するため、ゴミ箱に保持する案件は最大50件とします（古いものから順に完全削除されます）。
- **保存場所**: 削除済み案件は「管理画面」の中に「削除済み案件」タブを新設して管理します。

## 変更計画

### 1. 状態管理の拡張
- `this.state.deletedProjects` 配列を新規追加します。
- データの保存・読み込み・整合性チェック処理を更新し、この配列が正しく維持されるようにします。

### 2. UIの追加 (index.html)
- メンバー管理モーダル内に「削除済み案件」タブを追加します。
- ゴミ箱を表示するための領域を確保します。

### 3. ロジックの実装 (app.js)
- **削除処理の更新**: `delPj` 実行時、`deletedAt` 情報を付与し、かつ保持件数が50件を超える場合は古いデータから削除するロジックを追加します。
- **復元・完全削除処理の修正**: 実行直後に即座に表示が更新されるよう、`renderMemberBody('deleted')` の呼び出しを修正します。
- **削除済み一覧の改善**: 削除日時列を追加し、削除した順番で表示されることを保証します。

## 検証プラン

### 自動テスト (ブラウザ)
- [ ] 案件を削除し、メインリストから消え、ゴミ箱タブに出現することを確認。
- [ ] ゴミ箱で「復元」ボタンを押し、メインリストの元の位置（または最後尾）に戻ることを確認。
- [ ] 復元後に工程表の描画が正しく更新されることを確認。

### 手動確認
- タグや担当者の情報が削れることなく復元されるかを確認。
