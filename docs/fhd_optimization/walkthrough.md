# ワークスルー：FHD（1920x1080）UI最適化

プロフェッショナルな業務環境（1920x1080ディスプレイ）において、最高の視認性と操作性を提供するためのUIアップグレードを実施しました。

## 実装内容

### 1. グローバル・レイアウトの再定義
- **広大な横幅の活用**:
    - サイドバー幅: 260px → **284px**
    - 日次列幅: 220px → **240px**
- **縦解像度の最適化**:
    - 工程行の高さ: 100px → **120px**
    - 全体のベースフォントを `16px` を基準に微増させ、大画面での可読性を向上。

### 2. 詳細パネル（Detail Panel）の大型化
- 幅を `420px` から **`480px`** へ拡張。
- 納期（Deadline）や重要項目のフォントウェイトとサイズを上げ、視覚的な優先順位を明確化。

### 3. 製番リスト（Project View）の「Excel＋モダン」化
- **ヘッダー固定 (Sticky Header)**: 縦に長いリストでも、常に項目名が一番上に表示。
- **列幅の最適化**: 11列のデータをFHDの1920pxに収める最適なバランスに調整。
- **入力環境の改善**: セル内の `textarea` のサイズ感とフォントを、データ入力作業に最適な値に調整。

### 4. 工程バーの表現力強化
- バーの中に **[No] 指令書No : 製品名** の形式で情報を表示。
- 文字が小さくなりすぎないようパディングとフォントを調整。

## 変更ファイル

- [style.css](file:///c:/Users/kohei/.gemini/antigravity/scratch/web_project_manager/style.css): 定数、タイポグラフィ、テーブルヘッダー固定、パネル幅の変更。
- [app.js](file:///c:/Users/kohei/.gemini/antigravity/scratch/web_project_manager/app.js): `drawBars` 描画ロジックの調整（最小高さ120px、テキスト生成）。

## 検証結果
- 1920x1080環境において、左右・上下の余白バランスが美しく、かつ大量の情報を一度に視認できることを確認。
- 最大化した製番リストにおいて、データが重なることなく全項目が表示されることを確認。
