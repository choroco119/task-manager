# エクスポート・インポート機能の改善

データ出力時のファイル名指定の変更と、インポート時のデータ欠落（特別項目）を修正します。

## 変更内容

### 1. インポート処理の修正 (`applyData`)
現状、`applyData` メソッドにてデータを読み込む際、`specialItems`（休暇・出張などのマスタデータ）が考慮されていませんでした。これを追加し、ユーザーがカスタマイズした特別項目も正しく引き継がれるようにします。

### 2. エクスポートファイル名の変更 (`exportData`)
出力時のファイル名を、現在表示している工程の期間（月曜日〜日曜日）に基づいた形式に変更します。
- 形式: `工程データ(YYYYMMDD-YYYYMMDD).json`
- 例: `工程データ(20260406-20260412).json`

## 実施手順
1. `app.js` の `exportData` 内で、`this.state.monday` から開始日と終了日を計算し、ファイル名として構成するロジックを追加します。
2. `app.js` の `applyData` の `Object.assign` 部分に `specialItems` を追加します。

## 検証項目
- [ ] エクスポートを実行し、ファイル名が表示中の期間と一致することを確認。
- [ ] カスタマイズした「特別項目」が含まれるデータをエクスポートし、別の環境でインポートした際に正しく反映されることを確認。
