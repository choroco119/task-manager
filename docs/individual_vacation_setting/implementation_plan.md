# 特別項目の管理機能追加計画

休暇や出張などの「特別項目」の選択肢（バッジ表示）を直打ちから変更し、管理画面から自由に追加・編集・削除できるようにします。

## 背景
現在、予定割り当てモーダルの「休暇・特別項目」タブに表示されているバッジ（休暇、出張、会議、欠勤）はHTMLファイルにハードコーディングされています。これをマスターデータ化し、タグやチームと同様に管理画面（メンバー管理モーダル）から設定を行えるようにします。

## 提案する変更内容

### 1. データ構造の拡張 (`app.js`)
*   `app.state.specialItems` を追加します。
*   既存データがない場合は、初期値として現在の4つ（休暇、出張、会議、欠勤）をセットします。

### 2. 管理画面の拡張 (`index.html`, `app.js`)
*   **タブの追加**: `index.html` の `memberModal` 内に「特別項目」タブを追加します。
*   **描画ロジックの追加**: `app.js` の `renderMemberBody` 関数に、特別項目の追加・一覧表示・削除を行うための専用UI（テーブルと追加フォーム）を構築する処理を追加します。
*   **CRUD関数の追加**: `addSpecialItem`, `upSpecialItemN` (名前変更), `delSpecialItem` などの連携関数を実装します。

### 3. モーダル内の動的描画 (`index.html`, `app.js`)
*   **コンテナの設定**: `index.html` の `assign-s-sec` 内にあるハードコーディングされたバッジ群を削除し、空のコンテナ `id="specialItemContainer"` を配置します。
*   **描画の連動**: `app.js` の処理内で、モーダル起動時や `render` タイミングで `specialItemContainer` の中に管理画面で設定されたバッジを動的に生成します。

## ユーザー・レビュー確認事項

> [!IMPORTANT]
> **初期データの移行について**
> この機能を追加した初回の読み込み時に、システム側で自動的に既存の「休暇」「出張」「会議」「欠勤」の4つをマスターデータとして登録し、すぐに使える状態にする想定です。
> もし別の文字を初期値としたい場合はお知らせください。

## 検証プラン
*   管理画面（歯車アイコン）を開き、「特別項目」タブが追加されていることを確認する。
*   特別項目を新規追加・名称変更・削除し、データが反映されることを確認する。
*   工程表上のセルを右クリックし、「休暇・特別項目」タブの中に、設定した通りのバッジが表示され、クリックでテキストボックスに値が入ることを確認する。
