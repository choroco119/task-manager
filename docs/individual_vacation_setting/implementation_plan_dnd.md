# Engineering Grid: ドラッグ＆ドロップ（D&D）移動・並び替えの実装計画

予定（バー）をドラッグして、別の曜日や担当者に直感的に移動・並び替えができるようにする改修計画です。

## 課題の分析とアプローチ
現在、設定済みの工程を別の日に移動するには、「バーをクリック → 曜日のセレクトボックスを変更 → 保存」というステップが必要です。
これを、**HTML5標準の Drag and Drop API** を用いて直感的に操作できるようにします。

1. **バーのドラッグ化 (`draggable=true`)**: 全てのバーをドラッグ可能にします。
2. **ドロップ先の整備 (`ondrop`)**: グリッドの全セル（`.click-cell`）をドロップ可能領域とし、ドロップ先の曜日（`idx`）と担当者（`staffId`）を識別できるようにします。
3. **期間の保持**: 複数日に跨るバー（例: 月〜水 の3日間）をドラッグして木曜日にドロップした場合、期間を維持したまま移動（木〜土）するように計算します。
4. **縦の並び替え**: 同じ日に複数の予定が重なっている場合の「並び順（表示の段）」を入れ替えるため、データ構造に微調整を加えるか、ドロップ位置によって並び順（配列内の順番）を更新する処理を入れます。

## 提案する実装手順

### [MODIFY] `app.js` (描画ロジックとイベント管理)
#### 1. バー生成の拡張 (`drawBars`)
- 作成する `.bar` 要素に `draggable = true` を付与。
- `ondragstart` イベントを追加し、ドラッグ開始時に「自身のID」と「期間（日数）」のデータを `e.dataTransfer` に格納します。
- 操作中の視覚表現として、ドラッグ中はバーを半透明にします。

#### 2. ドロップ先（セル）の拡張 (`renderGrid`)
- `.click-cell` に `data-day` と `data-staff` 属性を付与し、座標を特定できるようにします。
- `ondragover` イベントでデフォルトの動作をキャンセルし、ドロップを許可します。
- ドロップ時の視覚フィードバック（ホバー中のハイライト等）を追加します。

#### 3. 移動・並び替えロジックの実装 (`handleBarDrop`)
- ドロップ時に実行される関数を新設します。
- `e.dataTransfer` からタスクIDを取得し、対象のデータを検索。
- 新しい担当者 (`staffId`) と、ドロップしたセルの曜日を新しい `startIdx` として更新します。
- `endIdx` は「新しい `startIdx` ＋ 期間（日数）」として再計算させます（土日をはみ出さないように補正処理も追加）。
- 同じセル内にドロップした場合は、既存のタスクとの順番（配列のインデックス）を入れ替えることで「並び替え」を実現します。

## オープンクエスチョン（ご確認事項）

> [!CAUTION]
> **ドラッグの挙動について**
> 1. 担当者（横の行）を跨いだ移動も許可してよいでしょうか？（Aさんの月曜日の仕事を、ドラッグでBさんの火曜日に渡すなど）
> 2. 3日間の予定を「金曜日」にドロップした場合、はみ出す部分（土曜・日曜以降）はどう扱うのが理想でしょうか？（土日で切って縮めるか、強制的にそのまま保存するか）
