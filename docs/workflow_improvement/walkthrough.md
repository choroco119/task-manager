# 工程入力ワークフローの改善（製番リスト直行・自動保存）完了報告

ユーザーがカレンダー上のセルをクリックした際の挙動を最適化し、製番の選択から工程追加までの手順を大幅に短縮しました。

## 実施内容

### 1. 新規入力時のダイレクト接続（`showA` の改善）
- カレンダーの空いているセルをクリックした際、これまでの割当モーダルを表示せず、**直接「製番リスト」画面を開く**ように変更しました。
- クリックされた場所（曜日と担当者）はバックグラウンドで保持されます。

### 2. 製番リストの全画面（最大化）表示
- 案件を選択しやすいよう、製番リストウィンドウをブラウザが許可する最大サイズ（`screen.availWidth/Height`）で、画面の左上（`0,0`）を起点に開くように調整しました。

### 3. 未選択からの自動保存ロジック（`selectPjFromSub` の拡張）
- 別ウィンドウの製番リストで「選択」ボタンを押すと、元の画面の保持されていたコンテキスト（クリックした場所）に対して、**工程が自動的に追加・保存**されます。
- これにより、モーダルを開いて確定ボタンを押す手間がなくなりました。

## ユーザーフローの比較

| 項目 | 以前のフロー | 新しいフロー |
| :--- | :--- | :--- |
| **ステップ 1** | セルをクリック | セルをクリック |
| **ステップ 2** | モーダルが表示される | **製番リスト（全画面）が開く** |
| **ステップ 3** | 「別窓から選ぶ」をクリック | 工程を選択して「選択」をクリック |
| **ステップ 4** | リストから選択して戻る | **（自動的に工程が追加される）** |
| **ステップ 5** | 「確定」ボタンをクリック | - |

---
> [!NOTE]
> **既存工程の編集について:**
> すでに配置されている工程バーをクリックした場合は、従来通り「割当モーダル」が開きます。これにより、期間の変更や工程の削除といった編集操作をこれまで通り行うことができます。
