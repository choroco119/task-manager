# 製番リストのタグ順ソート機能の実装計画

製番リストの順序を、タグのマスター登録順に基づいて一括で並び替えられるようにします。これにより、同じタグの案件をグループ化して管理しやすくなります。

## ユーザー要求事項の整理

- [x] **ソート順**: タグのマスター登録順に従う。
- [x] **未設定案件**: タグが設定されていない案件は最後尾に配置する。

## 変更内容

### app.js (ロジック実装)

#### [MODIFY] [app.js](file:///c:/Users/kohei/.gemini/antigravity/scratch/web_project_manager/app.js)
- `sortPjByTag()` メソッドを追加します。
- このメソッドでは `this.state.projects` 配列を以下の優先順位でソートします：
    1. タグのマスター配列（`this.state.tags`）内でのインデックス順。
    2. タグ未設定の案件は常に後ろ。
- ソート実行後、`validateIntegrity()` を呼び出して全案件の `No.` を振り直し、表示を更新します。

### index.html (UI構成)

#### [MODIFY] [index.html](file:///c:/Users/kohei/.gemini/antigravity/scratch/web_project_manager/index.html)
- 製番リストの検索バー（`pj-search-bar`）右側に、「タグ順で並び替え」ボタンを追加します。
- これにより、ユーザーの任意のタイミングでリストをタグ基準に整列させ、案件番号（No.）を確定させることができます。

## 検証プラン

### 自動テスト (ブラウザ操作)
- [ ] 複数のタグを登録し、タグ順が異なる案件を複数作成。
- [ ] 「タグ順で並び替え」ボタンをクリックし、リストがタグマスターの順序通りに並ぶことを確認。
- [ ] タグ未設定の案件が必ずリストの最後に来ることを確認。
- [ ] 並び替え後、`No.` が 1 から順に正しく採番し直されていることを確認。

### 手動確認
- タグマスターの順序を（▲/▼ボタンで）入れ替えた後、再度並び替えを実行して、新しいマスター順が反映されることを確認。
