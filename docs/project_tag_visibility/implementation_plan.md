# 製番リストのタグ境界の視認性向上計画

タグごとに案件が並んでいる際、タグの切り替わりがひと目で分かるように、グループごとの見出し行（セパレーター）を導入します。

## 変更内容

### app.js (描画ロジック)

#### [MODIFY] [app.js](file:///c:/Users/kohei/.gemini/antigravity/scratch/web_project_manager/app.js)
- `renderProjectBody` メソッド内で、案件をループ処理する際に直前の `tagId` を保持します。
- `tagId` が変化したタイミング（および最初の案件の前）で、タグ名を表示する特別な `<tr>` 行を挿入します。
- 検索（フィルタリング）実行中であっても、現在表示されている案件の中でタグが変わる箇所に適切に見出しを挟みます。

### style.css (デザイン)

#### [MODIFY] [style.css](file:///c:/Users/kohei/.gemini/antigravity/scratch/web_project_manager/style.css)
- `.tag-group-header` クラスを追加します。
- 見出し行はテーブルの全列を結合し、少し濃い背景色、太字、適度なパディングを持たせることで、セパレーターとしての役割を果たします。

## 検証プラン

### 自動テスト (ブラウザ操作)
- [ ] 複数の異なるタグを持つ案件を作成し、タグ順に整列させる。
- [ ] リストを表示し、タグの境目に「タグ名」が表示された見出し行が挿入されていることを確認。
- [ ] フィルタリングを行った際も、残った案件のタグ境界で正しく見出しが表示されることを確認。

### 手動確認
- タグ未設定の案件グループについても「タグ未設定」という見出しが表示され、他と区別できることを確認。
