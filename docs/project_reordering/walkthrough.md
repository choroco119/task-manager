# 並び替え機能（No.編集による割り込み）の実装完了報告

製番リストの「No.」を直接書き換えることで、直感的に案件の順序を入れ替えられる機能を実装しました。

## 実施内容

### 1. 製番リストの「No.」を編集可能に
製番リストのテーブルにおける `No.` 列を、数値入力フィールド（`<input type="number">`）に変更しました。
- 入力値を変更して `Enter` キーやフォーカスを外す（changeイベント）ことで、即座に並び替えが実行されます。
- 入力された番号の位置に案件が「割り込み」で移動し、前後の案件は自動的にシフトします。

### 2. デザインの調整
`style.css` に `.no-edit` クラスを追加しました。
- 表のデザインを損なわないよう、背景を透過させ、ホバー時やフォーカス時のみ枠線が表示されるようにしています。
- スピナー（上下矢印）を非表示にし、数値が見やすいように中央揃えに配置しました。

### 3. 並び替えロジックの実装
`app.js` に `reorderPj` メソッドを追加しました。
- 配列操作（`splice`）を使用して、指定された位置への案件移動を実現しています。
- 移動後、`validateIntegrity` を実行することで、全案件の `No.` を正しい連番に更新します。

### 4. 工程表への自動反映
`reorderPj` の最後で `this.render()` を呼び出すことで、メイン画面の工程表を再描画するようにしました。
- これにより、工程表のバーの中に表示されている `#No.` も最新の番号に同期されます。
