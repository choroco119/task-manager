# バー端部ドラッグによる期間変更機能の実装計画

ドラッグ＆ドロップ（D&D）による「移動」機能に加えて、バーの左右の端（エッジ）をドラッグすることで「期間のみを延長・短縮（リサイズ）」できる機能を実装します。

## 実装アプローチ

1. **リサイズ用ハンドルの追加 (`app.js`, `style.css`)**
   *   描画される各バー（`.bar`）の内部に、左端と右端のそれぞれに透明な「掴める領域（リサイズハンドル）」を配置します。
   *   マウスポインタが端に近づくとカーソルが左右矢印（↔）に変わり、直感的に掴めることを示します。

2. **ドラッグ操作の計算 (`app.js`)**
   *   ハンドルを掴んだとき（`mousedown`）の座標を記録します。
   *   ドラッグ中（`mousemove`）に移動したピクセル数を、グリッドの1マス（1日分）のピクセル幅で割り算し、「何日分動かしたか（+1日、-2日など）」を計算します。
   *   バーの見た目をリアルタイムで変化させ、視覚的なフィードバックを提供します。

3. **データの更新と保存 (`app.js`)**
   *   マウスを離したとき（`mouseup`）に、元々の「開始日」または「終了日」に対して動かした日数を加減算し、データを更新して保存します。
   *   ※ 日・月を跨ぐような異常値（0未満、6より大きい値）にならないように、自動で0(月曜)〜6(日曜)の間に収まるよう補正（制限）をかけます。

## イベントの競合回避策
現在 `.bar` には以下の機能が相乗りしています。
- 単純なクリック → モーダルを開く
- 全体のドラッグ → 今回実装したD&Dでの移動
- （NEW!）端のドラッグ → リサイズ

端を掴んだ時は「単純クリック」と「全体移動」が発動しないよう、イベントの伝播をキャンセル（`stopPropagation` / `preventDefault`）する高度な制御を組み込みます。
