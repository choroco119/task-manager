# リファインメントおよび機能拡張の実装計画

いただいた6点の改善ご要望について、以下の通り実装計画を作成いたしました。

## 実装内容

### 1. マウスオーバー詳細の保持
*   **対応**: 予定バーからマウスを離した際に詳細パネルを消去する処理（`onmouseout`）をなくします。
*   **結果**: ほかのバーにマウスを乗せる（上書きされる）まで、右側の詳細パネルがずっと表示され続けるようになります。

### 2. バーの表示テキストの変更
*   **対応**: 現在の「客先 / 製品名」から、「指令書No. / 製品名 / タグ名」の表示に切り替えます。

### 3. 個人合計工数の重複排除（名寄せ計算）
*   **対応**: 担当者の右側に表示されている合計工数（h）について、同じ人が同じ製番を週の複数の日（例：月と木）に分けて持っている場合は、「2回」足すのではなく**「その製番の工数を1回だけ」**加算するように計算ロジック（`Set`を利用）を修正します。

### 4. ドロップ時の「コピー」機能
*   **課題と代替案**: ブラウザ標準のドラッグ＆ドロップ機能（HTML5 D&D API）は、OSとブラウザの制約により**「右クリックでのドラッグ」を正しく認識できない（無反応になる）ケースが多発**します。
*   **提案**: 操作の確実性を担保するため、ExcelやOSのファイル操作などと同じ標準的な**「Ctrlキー（またはShift/Altキー）を押しながらドラッグ&ドロップ」すると、移動ではなく『コピー』になる**仕様で実装したいと考えます。

### 5. 「削除」ボタンを「解除」にリネーム
*   **対応**: 予定割当モーダル（右クリックで出る画面）にある赤い「削除」ボタンの設定や内部機能はそのままに、表示名だけを「解除」に変更します。特別項目や予定を無かったことにする操作は引き続きこれで行えます。

### 6. ゴミ箱連動の「完了」ボタンの新設
*   **対応**: モーダルの「解除」の横に、新しい「完了（製番を削除）」ボタンを設置します。
*   **挙動**: このボタンを押すと以下の2つが同時に実行されます。
    1. そのセルの予定を解除（消去）する。
    2. 対象の「製番」自体をプロジェクト一覧から削除し、管理画面の「削除済み案件（ゴミ箱）」へ送る。
*   ※「特別項目（休暇など）」を開いた場合は、対象の製番が存在しないため「完了」ボタンは非表示となるよう制御します。

## オープンクエスチョン（ご確認事項）

> [!WARNING]
> **ドラッグで「コピー」にする操作について（項目4）**
> 上述の通り、右クリックドラッグはブラウザ側の制限が強いため、代わりに**「Ctrlキーを押しながらドラッグ（またはドロップ）する」ことでコピーになる**という実装方針で進めてよろしいでしょうか？

上記をご確認いただき、「Ctrlキーで進めてよいか」等のご返答をいただけましたら、すべて一括で実装いたします！
